Presentation HTML5 audio API

This following chapter present the HTML5 audio API, it is a part of the course "HTML5: Web Design Fundamentals" by Michel Buffa. 

Introduction

HTML5 audio defines several layers:

  • The <audio> element, similar to the <video> element in its use and API.
  • The "Audio data and Web Audio API" dedicated to musical applications defines real-time sound processing, real time effects, access to samples, etc. Still experimental, presented briefly in this course.
  • Audio for games, still very experimental: addresses problems such as "low cpu and ram use", "uncompressed shot sounds that can loop", etc. 

This course will not talk about the third point.

The JavaScript API (properties, methods, events) of the <audio> element is just a "reduced" version of the the <video> element one, and we'll only address the differences and peculiarities.

The <audio> element

Just as for the video element, in this example we used the controls attribute so that the play/stop, time, volume and progress widgets will render. Notice the other similarities: between the <audio>...</audio> we added a text message that is displayed if the web browser doesn't support the <audio> element, and we used several <source>..</source> elements that link to different audio formats for the same file. The browser will use the first format it recognizes.

The <audio> element and the DOM

The <audio> element has methods, properties/attributes, and events that can be manipulated from JavaScript and the DOM API:
  • Methods for controlling the behavior - like play(), pause(), etc..
  • Properties (duration, current position, etc.), some that we can read/write (like volume), and others that are read only (like encoding, duration, etc)
  • Events are generated during the life cycle of the element and can be processed using JavaScript callbacks. We can also send events to control the audio player.
Just as any HTML element, the <audio> element can be manipulated / created using the DOM JavaScript API:

var audio = document.createElement('audio');
audio.src = 'audio.oga';
audio.controls = true;

Or also using a simpler method:

Attributes of the <audio> element

There are different attributes you can use with the html <audio> element. They are a subset of those available for the <video> element.

  • src: source of an audio stream
  • controls: if this boolean attribute is present, the browser displays its own controls for audio playback and volume.
  • autoplay: tells the browser to start playing the audio stream automatically as soon as the page is ready.
  • autobuffer: tells the browser to begin downloading the audio stream immediately, anticipating that the user will play the audio stream (this part of the specification is currently evolving and subject to change.)
  • loop: another boolean attribute that indicates to play the audio stream in loop mode (start again when finished).

Not all the attributes are supported by all browsers yet, but the most common ones are. The autoplay and autobuffer attributes are not recommended if your web site targets mobile applications, as it may consume bandwidth even if the user is not interested in listening to the proposed audio stream.

 

Support in current browsers / codec support

The <audio> element is-well supported in all major browsers, including on smart phones/tablets.

See an up-to-date version of this table: http://caniuse.com/#feat=audio

 

Just as for the <video> element, support for the different audio codecs is not the same in all browsers. There is a popular test page for audio formats you can try with different browsers: http://textopia.org/androidsoundformats.html, and a nearly up-to-date table available here: https://developer.mozilla.org/en-US/docs/Media_formats_supported_by_the_audio_and_video_elements. 

Javascript API for the <audio> element

The JavaScript API gives you powerful tools to manipulate the audio element. Indeed the audio object that can be manipulated through JavaScript provides many properties, methods, and events.

The complete list of events can be found at the W3C specification page. Another interesting resource is this blog post that explains in details how each event works (and indicates when an event is audio or video only): http://www.htmlgoodies.com/primers/html/article.php/3883356.

The list of properties for the <audio> element is similar to that of the <video> element and can be found at the HTML5 Video Events and API of the W3C. 

Most of the examples provided in the section about the <video> element API can be directly adapted for <audio> elements, like the examples that show how to manage a playlist, detect the end of a video, etc. Since both <audio> and <video> share the Media Events specification, they also share the same API for controlling behavior (play, pause, etc). As a result, adapting the video examples to audio is straightforward.

<!DOCTYPE html>

<audio id="player" controls="controls">
<source src="http://www.w3schools.com/html/horse.ogg" type="audio/ogg" />
 <source src="http://www.w3schools.com/html/horse.mp3" type="audio/mp3" />
 Your browser does not support the audio element.
</audio>
<div> 
<button onclick="document.getElementById('player').play()">Play</button> 
<button onclick="document.getElementById('player').pause()">Pause</button> 
<button onclick="document.getElementById('player').volume += 0.1">Vol+ </button> 
<button onclick="document.getElementById('player').volume -= 0.1">Vol- </button> 
</div>
</body>
</html>

This example uses the JavaScript API for controlling the play/pause/volume of the audio stream being played.
